## 当前里程碑：v0.1.x 养成系统

### 🟢 Phase A — 自发动作扩展（v0.1.0）

- 启动后 `waving` 打招呼（1.5s 后，2200ms）
- 单击洗牌池加入 `jumping`，扩为 5 项
- 鼠标快速接近触发 `spotted`（240px、≥ 0.6px/ms、10s 冷却）

### 🟢 Phase B — 数值养成系统（v0.1.0）

- 4 项数值 `hunger / cleanliness / mood / health` 接入衰减循环
- 阈值 < 30 时触发对应危险态 mood（优先级 health > hunger > cleanliness > mood）
- 用户互动 / 喂食 / 洗澡 / 吃药对应回补
- 持久化到 `userData/ram-care-stats.json`，离线衰减封顶 8h
- 右键菜单"查看状态" + 托盘菜单"今日状态"

### 🟢 节奏调慢 + 睡觉菜单（v0.1.0 后追加，未发版）

- 用户反馈"状态切换太频繁" → 把 walk/ambient 周期翻倍，把 ambient 池里 `idle` 加权
- 说话气泡 60s → 5min
- 菜单加"睡觉"（持久睡眠，任意交互唤醒）
- 详细决策见 [iterations.md "节奏调慢 + 睡觉菜单"](iterations.md)

### 🟡 Phase B 收尾 — 体感验收

待真人长期使用后验证：

- 离线 1 晚 / 1 周末后回来，数值是否合理（不死、但有需求）
- 危险态是否反复弹出造成打扰
- 菜单条件显示（< 60 出现）是否符合发现性预期
- 状态面板气泡位置是否被 Codex 气泡遮挡
- 新节奏（1-2 次/分钟）是否仍然太频繁，是否需要"安静模式"开关

## 下一里程碑候选

### 🔵 v0.2.x — 偏好设置 & 性格

- 性格选项（活泼 / 安静 / 黏人 / 学习陪伴）
- 衰减速率、睡眠等待时间、是否置顶、开机启动等用户可调
- 数值面板默认显示开关

### ⚪ v0.3.x — 数值驱动提醒升级

- 当前提醒文案池是固定的，升级为基于数值/上下文判断的动态提醒
- 接入 API（可选），让"该喝水/休息/活动"等提醒更智能

### ⚪ v0.4.x — 性格层 & 新状态

- 状态机之上加一层性格 modifier
- 新状态候选：`wake`（睡眠唤醒过渡）、`curious`、`shy`、`bump`、`thirsty`
- 每个新状态需绑定独立素材，避免状态名扩张但视觉无差

### ⚪ vX — 发布与合作准备

- macOS `.app` / `.dmg` 完整签名
- 开机自启
- 版权与素材合规

## 已废弃 / 暂停

### ⚫ 蓝色果冻形象替换（暂停）

- 现状：[scripts/generate-ram-azure.mjs](../scripts/generate-ram-azure.mjs) 已可生成蓝色果冻候选
- 决策：当前阶段先把交互层做稳，视觉风格不动；如果未来要切换，需要批量重生成所有核心态保证风格统一

### ⚫ 悬停浮出状态面板

- 替代方案：右键菜单"查看状态"（v0.1.0 已采用）
- 决策原因：悬停浮出会干扰"悬停仅唤醒"的现有语义，且鼠标路过频繁触发
